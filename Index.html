  1. Общий анализ требований

    2. Сбор и анализ требований, предъявляемых к итоговому продукту.

    • Вход и регистрация в приложении
      с указанием описания о собаке и требованиям к выгулу
    • Определение места положения пользователя для поиска ближайших кандидатов
      по выгулу собаки (выгульщиков)
    • Создание функции удаленного отслеживания процесса выгула
    • Создания спопсобов оплаты за услугу выгула

    Полученные требования реализуются в концепции проекта.

  3. Разработка приложения

    • Разработка продукции по проектной документации

    В результате разработки получаем готовое приложение для тестирования.

  4. Тестирование приложения

    • Проверка тестироввщиками разработанного продукта

    На данном этапе проиходит проверка соотвествия реализованного функционала описанным требованиям.

    Составляется тестовый отчет, где указанно что приложением имеет интуитивно непонятный интерфейс.

  5. Техническая поддержка
    6. Наблюдение за функционированием приложения и устранение появляющихся недостатков:

    — Поддежка пользователей

    В конечном итоге решение о внесении доработок и новый проход этапов жизненного цикла приложения с вновь принятыми изменениями.
